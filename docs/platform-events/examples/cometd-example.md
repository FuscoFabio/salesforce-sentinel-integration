# Caso d'Esempio: CometD Streaming

Esempio completo di implementazione CometD Streaming per integrazione Platform Events con Azure Sentinel.

## Scenario

Un'organizzazione ha bisogno di:
- Connessione persistente WebSocket
- Push real-time continuo
- Volume eventi continuo

## Requisiti

- Licenza Salesforce Enterprise
- Azure Subscription
- Azure Function (Long-Running)
- Competenze avanzate sviluppo
- Tempo setup: 3-5 giorni

## Implementazione Completa

Vedi la guida dettagliata: [Implementazione CometD Streaming](../implementation/cometd-streaming.md)

## Caratteristiche

- Connessione persistente
- Push real-time
- Efficienza WebSocket
- Long-running process

## Risultati Attesi

- Eventi disponibili in Azure Sentinel in < 1 minuto
- Connessione persistente
- Push continuo eventi
